# NanoHarness 核心组件索引

这是一份经常查阅的结构索引，用来快速定位 tool execution、validation、permission、
sandbox 和 observability 背后的 runtime 组件，避免每次都扫描整个仓库。

> 快照说明：如果仓库在本文写入后发生变化，修改代码前仍需确认路径和签名。

## Tool Registry、Router 与 Wiring

- `agent_forge/tools/registry.py`：`ToolRegistry`，名称到工具的主 registry 和 execution
  boundary。重点方法：`register()`、`schemas()`、`execute()`、`_validate_arguments()`。
- `agent_forge/tools/tool_router.py`：`ToolRouter`，根据 intent、risk/mode metadata、
  read-only 表述和 active skill tool name 筛选每个 turn 可见的 tool schema。
- `agent_forge/runtime/wiring.py`：`build_registry()`，构造 `WorkspaceSandbox`，注册
  list/read/write/grep/patch/run/git/diagnostics/ask-human 和配置的 MCP tool。
- `agent_forge/tools/ask_human.py`：只声明 control signal；直接执行会 fail closed，
  persistence 和 stop 由 `AgentLoop` 负责。
- `agent_forge/tools/base.py`：定义带 `schema()`、`execute()` 的 `Tool` 基础契约。
- `agent_forge/tools/mcp_config.py`、`agent_forge/tools/adapters/mcp_style_adapter.py`、
  `agent_forge/tools/mcp_stdio.py`：将 MCP/external tool 接入本地 registry。
- `agent_forge/skills/registry.py`：`SkillRegistry` 是相邻的 capability registry，不是
  core tool registry；被选 Skill 可通过 `tool_names` 扩大 router 访问范围。

## Validator、Schema Validation 与 Guardrail

- `agent_forge/tools/registry.py`：`_validate_arguments()` 和 `_matches_type()` 在执行前
  校验 required argument 和 primitive type。
- `agent_forge/runtime/structured_output.py`：`StructuredOutputParser` 从模型文本提取
  JSON，校验 JSON-Schema-like subset，并构造 repair prompt。
- `agent_forge/runtime/llm_client.py`：`_parse_tool_calls()` 校验 provider tool-call
  argument 是 JSON object；`_tool_to_openai_schema()` 转换本地 tool schema。
- `agent_forge/safety/guardrails.py`：`input_guardrail()`、`output_guardrail()`、
  `tool_guardrail()` 分别处理风险输入、final-answer claim 和 unknown/repeated/malformed
  tool call。
- `agent_forge/safety/command_policy.py`：`check_command()` 根据 deny/allow rule 校验
  shell-like command。
- `agent_forge/skills/registry.py`：`SkillSpec.from_mapping()`、`_dict_field()`、
  `_list_field()` 校验 skill manifest。

## Permission 与 Approval Policy

- `agent_forge/safety/permission.py`：`PermissionPolicy`、`PermissionDecision` 是统一
  allow/ask/deny 决策层。Read 通常允许，write 需要询问，command 继续经过检查，
  network/delete/external action 被拒绝。
- `agent_forge/safety/command_policy.py`：`run_command` 的 command defense，阻断危险
  command，只允许安全 validation 和只读 git inspection。
- `agent_forge/runtime/hooks.py`：`ApprovalMode`、`PermissionHook`、
  `HookManager.default()` 将 permission policy 接入 runtime；mode 包括 `trusted`、
  `on-write`、`on-risk`、`locked`、`dry-run`。
- `agent_forge/runtime/tool_execution.py`：构造 `HookContext`，按固定顺序执行
  guardrail、permission、approval、operation ledger 和 tool，并记录 observation。
- `agent_forge/runtime/config.py`、`agent_forge/forge_cli.py`：提供 `approval_mode`、
  `auto_approve_writes` 等 runtime/CLI 开关。
- `write_file.py`、`apply_patch.py`、`run_command.py` 在副作用前还有 tool-level defense。

## Human Input 与 Continuation

- `agent_forge/runtime/human_input.py`：`HumanInputStore`，原子持久化
  pending/responded/cancelled 信息请求，包含 stable id、choice validation 和 terminal
  state protection。
- `agent_forge/runtime/agent_loop.py`：处理 pre-loop clarification；
  `agent_forge/runtime/tool_execution.py` 拦截模型 `ask_human`。
- `agent_forge/runtime/run_lifecycle.py`：统一记录 `waiting_human`、加载已有 response，
  并把 cancelled request 转成 terminal transition。
- `agent_forge/forge_cli.py`：`forge respond` 记录回答或取消；`forge resume` 将已回答
  question 注入 continuation context。
- `agent_forge/runtime/approval.py` 保持独立，因为 authorization 和 clarification 的
  语义不同。

## Sandbox、Workspace Boundary 与 Execution Environment

- `agent_forge/safety/sandbox.py`：`WorkspaceSandbox` 将 path 解析在 workspace root 下，
  阻断 traversal、external path 和 secret-like file。重点方法：`resolve_path()`、
  `is_sensitive_path()`、`ensure_safe_path()`。
- `agent_forge/runtime/wiring.py`：创建 shared sandbox 并注入内置 tool。
- `read_file.py`、`list_files.py`、`grep.py`、`write_file.py`、`apply_patch.py`、
  `run_command.py` 使用该 sandbox。
- `agent_forge/runtime/execution_environment.py`：`ExecutionEnvironmentConfig` 和
  `ExecutionEnvironment`，提供 local、isolated worktree、受限 OCI mode，以及 snapshot
  挂载、resource/network policy、command delegation、manifest/replay evidence、path
  validation、redaction。
- `agent_forge/runtime/git_workspace.py`：统一的 HEAD-relative binary diff/status 层，
  包含 untracked source file，仅排除 untracked runtime-owned `.agent_forge` artifact。
- `agent_forge/runtime/hooks.py`：`ExecutionEnvironmentHook` 在 tool 前执行 path/command
  restriction。

## Observer、Hook、Event 与 Trace

- `agent_forge/observability/trace.py`：`TraceRecorder`，append-only run event、metric
  collection 和最终 trace JSON writer。
- `agent_forge/observability/event.py`：`TraceEvent` typed event shape。
- `agent_forge/runtime/hooks.py`：`RuntimeHook`、`HookManager` 提供 pre-tool/post-tool/
  on-stop observer 和扩展机制，包括 permission、environment、secret redaction hook。
- `agent_forge/runtime/agent_loop.py`：负责 run/turn 阶段、context/tool routing、model call
  和 final answer event。
- `agent_forge/runtime/tool_execution.py`：负责 hook、permission、tool、observation、
  recovery 和 operation event。
- `agent_forge/runtime/run_lifecycle.py`：负责 checkpoint、human pause 和 stop event。
- `agent_forge/observability/metrics.py`、`usage_report.py`、`evidence.py` 消费 trace，
  生成 counter、人类可读 usage report 和可引用 evidence。

## 多 Agent 编排

- `agent_forge/multi_agent/coordinator.py`：顺序 Implementer/Reviewer/Verifier workflow，
  显式 artifact handoff，revision 有上限。
- `agent_forge/multi_agent/fanout.py`：纯 dependency batching 和 declared/actual overlap
  算法。
- `agent_forge/multi_agent/live_fanout.py`：validated JSON plan、disposable worktree
  worker、独立 AgentLoop/LLM/registry、scope gate、确定性 merge、增量 checkpoint、
  selective recovery 和隔离 finalizer。
- `examples/fanout-plan.sample.json`：适合第一次运行的安全双 worker 只读 plan。

## 推荐阅读顺序

1. `agent_forge/runtime/wiring.py`
2. `agent_forge/tools/registry.py`
3. `agent_forge/runtime/agent_loop.py`，第一遍只展开 `run`
4. `agent_forge/runtime/state.py`
5. `agent_forge/runtime/tool_execution.py` 和 `run_lifecycle.py`
6. `agent_forge/safety/permission.py` 和 `agent_forge/safety/sandbox.py`
7. `agent_forge/observability/trace.py`
8. `agent_forge/runtime/human_input.py`
9. `agent_forge/multi_agent/live_fanout.py`
10. `agent_forge/runtime/git_workspace.py`
